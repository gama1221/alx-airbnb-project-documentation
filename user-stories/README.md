## Objective
This document summarizes the core user stories derived from the use case diagram for the Airbnb Clone backend.

---

### Core User Stories

1. **User Authentication:** Guests can register, log in, and manage their profiles.  
2. **Property Management:** Hosts can add, update, and delete properties for booking.  
3. **Booking System:** Guests can book properties for specific date ranges.  
4. **Payments:** Guests can make secure payments and track payment status.  
5. **Reviews & Ratings:** Guests can leave reviews and ratings for properties.  
6. **Messaging:** Guests and hosts can communicate regarding bookings and property details.  
7. **Admin Features:** Admins can monitor and manage users, properties, bookings, payments, and reviews.
